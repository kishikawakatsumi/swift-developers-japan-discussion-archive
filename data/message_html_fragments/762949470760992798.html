notFoundは404にして
ほかの例外は全部500でも良さそう。
デフォルトだとroutesが無かったら↓が飛んでるらしい
struct RouteNotFound: Error { } extension RouteNotFound: AbortError { static var typeIdentifier: String { "Abort" } var status: HTTPResponseStatus { .notFound } }
AbortErrorはprotocolになっててstatusをもってるから
例外をAbortErrorにキャストしてstatusを調べて404かその他か決められそう。
