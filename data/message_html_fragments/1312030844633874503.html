Kuniwakさんが言っているのは、そのような健全性条件は容易に崩れるので、多くのケースで let にすることになるということじゃないでしょうか。  たとえば、健全性条件を考えると var age: Int は許されず、 let age: Int か var age: UInt にすることになると思います。Swiftにおいて一般的には非負整数であっても Int を使うことが推奨されているので（ UInt は簡単にオーバーフロー→クラッシュしますし）、 let age: Int を選ぶことになりそうです。 そう考えると、多くの Int 型のプロパティは let が求められるように思います。
