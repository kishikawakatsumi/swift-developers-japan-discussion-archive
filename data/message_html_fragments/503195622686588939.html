値型だとインスタンス共有が存在しないので、振る舞いが定まります。
これをドキュメントやコーディング規約（たとえば必ず防御的コピーをする）とかで管理しようとしても
巨大なコードベースでは次第にそれが破られてきて、そのうち何が起こるか確信できないコードの塊が生まれるんじゃないかと思います。
イミュータブルクラスだとそういう問題はないですが、
状態の更新が大変です。
class User { let name: String let age: Int ... } var user = User(name: "Swift", age: 4) user = user(name: user.name, age: user.age + 1)
 (edited)
インスタンスを作り変えないといけないです。
ミュータブルクラスなら楽なのに。
// ミュータブルクラス class User { var name: String var age: Int ... } let user = User(name: "Swift", age: 4) user.age += 1 // 楽
 (edited)
これを struct でやると
// struct struct User { var name: String var age: Int } var user = User(name: "Swift", age: 4) user.age += 1 // 楽
 (edited)
楽だけどインスタンス共有もされなくて挙動が定まってうれしい
