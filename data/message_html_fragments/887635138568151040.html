これが結局RC版でも直っていないので、TextFieldに指定している文字列リテラルが単なるStringと解釈されてしまって、ローカライズが効かなくなるパターンが増えそうな気がします
iOS15がavailableな場合とそうでない場合で選ばれるイニシャライザが変わる
struct ContentView: View { @State private var text = "" var body: some View { if #available(iOS 15, *) { TextField("foo", text: $text) // "foo"はLocalizedStringKey扱い } else { TextField("foo", text: $text) // "foo"はString扱い } } }
