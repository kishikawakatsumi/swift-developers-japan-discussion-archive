array.values(<, x) array.values(<=, x) array.values(>, x) array.values(>=, x)
の4種類を作りたくて、本当は第一引数の演算子を使って一つの実装で実現できればよかったんですが、それが難しかったので四つのメソッドのオーバーロードになっています。第一引数はどのオーバーロードが呼ばれるかを区別するためだけに使われており、オペレーションには使われていません。 Swift的には array.valuesGreaterThan(x) とかが正しい命名だと思いますが、コンテスト中の可読性（かんちがいしないこと、ひと目でわかること）が重要だろうと考えてこのような API にしました。なので、まずほしい API の形があり、それを実現するために意味のない演算子を渡すことになったという形です（そのために、わざわざ通常の >= 等とは別に演算子そのもののオーバーロードも宣言する必要がありました）。
