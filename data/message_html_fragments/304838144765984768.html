あ、そうだ。それなんですけど、書こうと思ってて忘れてました。 Twitter に書いた方が後から見やすいかもですが、 SimpleDomainError が throws じゃダメな理由がわかりました。
純粋な値がない Optional と Simple domain error が区別されることも意味がありますが、 Simple domain error が Recoverable error と区別されることにも意味がありそうです。
なぜなら、
struct SimpleDomainError: Error {}
は、当然ですが Simple domain error なので、エラーが発生したという情報しか持たないわけです。
