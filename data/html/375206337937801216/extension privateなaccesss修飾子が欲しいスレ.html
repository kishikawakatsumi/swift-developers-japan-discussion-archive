koher 09-Aug-21 01:27 PM
ローカル関数、論理的には正しいと思うんですが、
if isFoo { // すごく長いコード return bar } else { return baz }
とするよりも
guard isFoo else { return baz } // すごく長いコード return bar
が良いみたいな。 true, false のケースが対等な場合で。
ローカル関数で嫌だったのは、 prepareViews の先頭にローカル関数を書くと実際の処理までたくさんスクロールしなければならず、後ろに書いた場合、
extension ViewController { private func prepareViews() { prepareAView() prepareBView() prepareCView() // ここから先がローカル関数の宣言だけな保証がない func prepareAView() { // 長いコード } func prepareBView() { // 長いコード } foo() // もしここに何か処理が書かれていたら？スクロールしないとわからない。 func prepareCView() {} } }
みたいな可能性があり、結局長いメソッドのコードをスクロールしないといけなくなるなぁと。 (edited)
